about

Nansi is a utility and reference for working with the ANSI 256 color palette. Nansi can be sourced into your .bashrc, or imported into your style.scss/sass.

download

Nansi comes in a few flavors; all of which can be found at the repository on Github; where the subtle differences are explained.

options

Some useful options for using this page have been mapped to the keyboard numbers 1–5. Use them to toggle the background color, show hex values, increase the color size, change column spacing, and change row spacing.

Nansi also contains some built in functions to print the colors to stdout; just enter: nf256, or nb256 at the prompt; to see the foreground or background colors; respectively.

install

Nansi is like any other file you source ~/nansi into your .bashrc (or .file of choice), and @import "_nansi.scss"; into your style.scss.


source ~/.nansi.tp

source ~/.functions



@import “_nansi.scss”;

@import “_mixins.scss”;

usage

You can use Nansi to customize your prompt; just make sure you escape the colors to avoid prompt wrap: PS1="\[${f160}\]\u\W\[${stop}\]▻ ", and as expected in your functions, scripts, printfs, echos etc. Colors will supercede eachother, where as styles will continue until the stop is called.

The syntax is straight forward; f = foreground / b = background, followed by the reference # 0–255 e.g.,$f162 colors text 162, and $b143 colors background 143. In addition to the colors, you'll need: $stop — to stop a color or style. The rest of the styles are as follows: $uline — begin underline, $bold — bold text, $rvrs — reversed out text.


printf "${234}foreground 234${stop}"

foreground 234



printf "${f234}foreground 234 ${uline}underlined${stop}"

foreground 234 underlined



printf "${f234}foreground 234 ${b210}background 250${stop}"

foreground 234 background 250

Using it with SASS/SCSS is pretty straight forward. The variables/colors are mapped to their reference numbers, preceded by an underscore: 

    
    .class { 
color: complement($_121);

    }
    
    

.class { 

color: $_238;

border: 1px solid $_96;

}

notes

I use OSX 10.9.4, Iterm2, and BASH⁴ installed via Homebrew. My use of the palette is less than advanced; simple functions, variables, aliases, expansions, and substitutions. Everything works perfectly for me, within my small, modest environment, but it may not for another.

If your using OSX, the default Terminal.app, and iTerm.app (my favorite) both support 256 colors. You may need to go into preferences and set the declare/terminal type as xterm-256color. You can also check the number of supported colors in your current shell by typing tput colors at the prompt, which will return that info. Lastly, it's also recommended that you lookup man terminfo to see the full range of possibilities for your particular environment.
