NANSI

DESCRIPTION

Nansi simplifies working with the xterm-256 color palette. Nansi reduces style/escape-sequence bloat, by using a concise, referential syntax. In addition, Nansi has a nice modern, user-focused reference for working with the palette (you are here). Nansi was originally designed because I wanted the option to explore a variety of colors in my bash dotfiles/functions. It has since been extended into the preprocesser and swatch file formats.

SYNOPSIS

    KEYBOARD NUMBERS:
  1. : change the background from dark (default) to light.
  2. : switch between the reference numbers (default) to hex values.
  3. : change the color visibility from border (default) to background.
  4. : turn column spacing off (default) and on.

INSTALL

Nansi can be sourced into your .bashrc (or dotfile of choice), imported in your scss/sass stylesheet, or imported into programs that support the .ase (Adobe Swatch Exchange) format. For detailed install instructions, and example usage, refer to the readme.pdf

source ~/.nansi
@import “_nansi.hex.scss”;
@import “_nansi.hex.less”;

SH/BASH

The syntax is simple; a number 0–255 preceded by f(oreground), or b(ackground). In addition to the colors, there's a few additional options all of which are outlined below:

  • f(0–255) : set a foreground color
  • b(0–255) : set a background color
  • stop : stop a color or style
  • line : set underline
  • bold : set bold text
  • rvrs : set reversed text
  • ital : set italic text
printf "${f8}foreground eight${stop}"
foreground eight
printf "${f8}foreground eight ${line}underlined${stop}"
foreground eight underlined

SASS/SCSS

Again, the syntax is simple; A dollar sign, followed by an underscore (for legibility), then a reference number 0–255.

.class {
color: invert($_121);
border: 1px solid $_96;
}

LESS

Same as above, aside from the variable declaration. In this case an at-sign, followed by an underscore (for legibility), then a reference number 0–255.

.class {
color: @_121;
border: 1px solid @_96;
}
